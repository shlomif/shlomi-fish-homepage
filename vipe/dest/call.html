Call for Participants

The project is still young, and as we have a very large list of things to do, we hereby call you - programmers who wish to help and/or learn - to join us. There is work to do in kernel mode as well as in user-mode. There is also need for more documentation. And above all - there is work to do regarding testing - both manual and automated tests.

Among the activities that are worked on now (and in the near future) are:

  • Support for more system call types.
  • Support for other action types (e.g. suspending the process that matched a tracking rule's filter).
  • Support for 'process tree' logging (i.e. parent process of the process that invoked the syscall, parent of that process, etc, up to the top-level process) - to allow answering questions such as 'ok, so the '/bin/kill' command killed my process, but who the heck invoked it?'.
  • speed and locking optimizations (to reduce impact of rule matching on overall system performance).
  • Support for logging via a device file.
  • Support for FreeBSD.

NOTE: The above is just what we plan to do real soon now. There are larger plans for the long run, so keep tuned.

If you wish to participate in the project's development, please look at our code-page, and register to the development mailing list - details available at: http://lists.sourceforge.net/lists/listinfo/syscalltrack-hackers. You may also wish to read the 'DEVELOPING' file that comes with the source code, for information for developers. Finally, you might wish to do most of the development and basic testing using User-Mode Linux - a port of Linux to... the Linux kernel architecture ;). This has saved us a lot of time recovering from kernel Oops.

The developers also hang out on the IRC channel #syscalltrack, on the Efnet IRC network. Come by and say hi sometime...
