What is it?

The syscall tracker allows you - the 'root' user - to track invocations of system calls across your Linux system. You specify rules that specify which system call invocations will be tracked, and what to do when a rule matches a system call invocation. Currently - only logging the invocation is supported, but in the future, you will be able to fail the system call (i.e. force it to return some error code), or suspend the process executing it (e.g. so you could attach a debugger to the process at that point, or so you could find who is the parent process of that process, etc.).
